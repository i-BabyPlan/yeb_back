登陆流程
    首先，前端传用户名和密码到后端
    后端校验用户名和密码
        有错误，回显重新输入提示
        正确，会生成jwt令牌，并且返回给前端，前端会的将这个令牌放入其请求头，后面任何请求都会懈怠jwt令牌，后端有拦截器，会对jwt令牌做相应的验证，通过
                                                                                                                            后才能访问对应的借口
                                                                                                                       不通过，说明jwt令牌失效了，或者用户名有问题，或者密码有问题，即不是合法登陆用户，将会阻止操作
    准备一个公用返回对象 放到pojo
